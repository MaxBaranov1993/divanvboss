# Прототип divanboss.ru — «как сейчас» vs «как улучшить»

React-прототип, чтобы показать разработчикам, какие правки нужны на сайте. Сравниваем три ключевые страницы в двух вариантах:

| Страница | Текущая версия | С улучшениями |
|---|---|---|
| Главная | `/current/home` | `/improved/home` |
| Карточка товара | `/current/product` | `/improved/product` |
| Оформление заказа | `/current/checkout` | `/improved/checkout` |

Корень `/` — лендинг со ссылками на все шесть страниц.

## Запуск

```bash
cd prototype
npm install
npm run dev
```

Откроется http://localhost:5173.

## Что изменено и почему

### Главная (`/improved/home`)

Источник правок: `analyses/2026-04-17-homepage-tech-audit.md`.

- **Добавлен H1** «Диваны от фабрики в Нижнем Новгороде». Сейчас H1 на главной отсутствует — критично для SEO (одна из причин, почему Яндекс-органика хуже Google).
- **Корректный `<title>` и `<meta description>`**. Сейчас description сбит и описывает страницу салонов, а не главную.
- **USP-полоса** (фабрика / 18 мес гарантии / рассрочка 0 % / возврат 14 дней) — снимает страх предоплаты в дорогой категории.

### Карточка товара (`/improved/product`)

- **«Купить в 1 клик»** (имя + телефон) — часть аудитории в мебели не готова к полной форме, но готова оставить контакт. У конкурента askona.ru это есть, у нас — нет.
- **Расчёт доставки по индексу** прямо в карточке — убирает тревогу «сколько накрутят при получении».
- **Sticky-колонка покупки** на десктопе — при скролле по описанию цена и CTA не уходят из поля зрения.

### Оформление заказа (`/improved/checkout`)

Источник правок: `reports/2026-04-21-top5-checkout-pravok.md` (все 5 правок реализованы).

1. **Адрес = одно поле** с мок-автоподсказкой DaData. Сейчас 10 полей — удар по мобильной конверсии.
2. **Стоимость доставки** считается после ввода адреса и видна в «Итого». Сейчас «оплачивается по факту» — демотиватор для дорогих покупок.
3. **«Оформить по телефону»** как альтернатива полной форме.
4. **Sticky-блок «Итого + CTA»** на десктопе + bottom-bar на мобиле. Сейчас при скролле CTA уходит наверх.
5. **Кросс-селл** «С этим покупают» + тумблеры доп-услуг (сборка, вывоз старой, подъём) — рост среднего чека.
6. **Email сделан опциональным** — часть ЦА 40+ почту не проверяет.
7. (Бонус) Плашка «начислим кешбэк» как у divan.ru.

## Как это устроено в коде

```
src/
├── main.jsx            — вход React
├── App.jsx             — роутер, лендинг
├── shared/             — VersionSwitcher, Placeholder
├── mock/data.js        — товары, категории, адреса
├── current/            — копия текущих страниц divanboss.ru
└── improved/           — те же страницы с правками
```

Шапка и футер в `improved/` импортируются из `current/` — визуально не меняются, меняется только контент внутри страниц.

## Ограничения прототипа

- Логотипы и фотографии товаров — placeholder-блоки (копирайт).
- DaData — мок-список адресов в Нижнем Новгороде, не реальное API.
- Отправка формы — `console.log`, без бэкенда.
- Мобильная адаптация — базовая, без попиксельной вылизки.
